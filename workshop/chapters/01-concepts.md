## Physical AI란?

<b>Physical AI</b>는 물리 세계에서 동작하는 인공지능입니다.

자율주행차, 로봇 팔, 보행 로봇, 드론 등이 여기에 해당하며, 기존 AI와 결정적으로 다른 점이 있습니다:

| | 기존 AI (LLM, 이미지 생성) | Physical AI (로봇) |
|---|---|---|
| <b>입력</b> | 텍스트, 이미지 | 센서 (IMU, 관절 인코더, LiDAR) |
| <b>출력</b> | 텍스트, 이미지 | 모터 제어 신호 (토크, 위치) |
| <b>실패 비용</b> | 없음 | <b>로봇 파손, 사람 부상</b> |
| <b>학습 환경</b> | 대규모 텍스트 데이터 | <b>시뮬레이션</b> + 실제 환경 |
| <b>물리 법칙</b> | 해당 없음 | <b>중력, 마찰, 충돌 필수</b> |

> <b>핵심 통찰</b>: Physical AI에서 <b>실패 = 물리적 피해</b>이므로, 시뮬레이션이 핵심입니다.

---

## 왜 시뮬레이션인가?

실제 로봇으로 학습하면:

```
1대의 로봇 × 실시간 동작 × 수만 번 시행착오 = 수개월 + 수천만 원 수리비
```

시뮬레이션으로 학습하면:

```
4,096대의 가상 로봇 × 100배속 × 병렬 실행 = 75분 + $12
```

이번 워크샵에서 실제로 경험할 수치입니다.

### Sim-to-Real Pipeline

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  시뮬레이션   │     │ Policy Export │     │  실제 로봇   │
│              │     │              │     │              │
│ 4096개 환경  │────▶│  ONNX / JIT  │────▶│ 보행 제어    │
│ GPU 병렬     │     │  (1.2 MB)    │     │ 실시간 추론  │
│ 75분 학습    │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
```

시뮬레이션에서 학습한 정책(신경망)을 ONNX/JIT 형태로 <b>export</b>하면, 실제 로봇에 탑재하여 동일한 보행 능력을 발휘할 수 있습니다.

---

## 강화학습(RL) 기초

이 워크샵에서 사용하는 학습 방법은 <b>강화학습(Reinforcement Learning)</b>입니다.

```
                    ┌─────────────┐
                    │ Environment │  (Isaac Sim: 4096개 동시 시뮬레이션)
                    │  - 지형 생성 │
                    │  - 물리 시뮬 │
                    │  - 충돌 감지 │
                    └──────┬──────┘
                           │
              관찰(Observation)│  보상(Reward)
           - 관절 각도/속도    │  - 잘 걸으면 +
           - 몸체 기울기       │  - 넘어지면 -
           - 발 접촉 상태      │  - 에너지 낭비 -
           - 명령 속도         │
                           ▼
                    ┌─────────────┐
                    │    Agent    │  (신경망: Actor-Critic)
                    │             │
                    │  Actor:     │ ← 관찰 → 행동(관절 토크) 매핑
                    │  [512→256→128]
                    │             │
                    │  Critic:    │ ← 관찰 → 가치(미래 보상 예측)
                    │  [512→256→128]
                    └──────┬──────┘
                           │
                     행동(Action)
                  - 12개 관절 토크
                           │
                           ▼
                    시뮬레이션 다음 스텝
```

### 핵심 요소

| 요소 | 설명 | 이번 워크샵에서 |
|------|------|-----------------|
| <b>Environment</b> | 로봇이 동작하는 가상 세계 | Isaac Sim (PhysX 5 물리 엔진) |
| <b>Observation</b> | 로봇이 감지하는 센서 데이터 | 관절 각도, 몸체 자세, 명령 속도 등 |
| <b>Action</b> | 로봇이 수행하는 동작 | 12개 관절의 목표 토크 |
| <b>Reward</b> | 행동의 좋고 나쁨 점수 | 속도 추적(+), 넘어짐(-), 에너지 낭비(-) |
| <b>Policy</b> | 관찰→행동 매핑 함수 | MLP 신경망 (Actor-Critic 구조) |
| <b>Algorithm</b> | 정책을 개선하는 방법 | PPO (Proximal Policy Optimization) |

## 이번 워크샵의 목표

<b>ANYmal-C</b> 4족 보행 로봇을 <b>거친 지형</b>(바위, 경사면, 계단)에서 걷게 만듭니다.

```
목표: "앞으로 걸어"라는 속도 명령 → 넘어지지 않고 정확하게 따라가는 보행 정책 학습

방법: PPO 강화학습
     - 잘 걸으면 +보상
     - 넘어지면 -패널티
     - 이걸 1.47억 번 반복 → 최적의 보행 패턴 발견
```

### ANYmal-C

[ANYmal-C](https://www.anybotics.com/anymal)는 스위스 ANYbotics社가 개발한 산업용 4족 보행 로봇입니다.

- <b>무게</b>: ~50 kg
- <b>관절</b>: 12개 (다리 4개 × 3관절)
- <b>센서</b>: IMU, 관절 인코더, LiDAR, 카메라
- <b>용도</b>: 산업시설 자율 점검, 위험 지역 탐사

## 소프트웨어 스택 이해

```
Layer 4: RSL-RL (PPO 알고리즘)           ← 학습 알고리즘
Layer 3: Isaac Lab v2.1.0               ← RL 프레임워크 + 환경 정의
Layer 2: Isaac Sim 4.5.0                ← 물리 시뮬레이션 + GPU 렌더링
Layer 1: NVIDIA Driver 580 + CUDA       ← GPU 기본 드라이버
Layer 0: Ubuntu 22.04 on EC2            ← OS
```

| 레이어 | 역할 | 핵심 기능 |
|--------|------|-----------|
| <b>Isaac Sim</b> | 물리 시뮬레이션 플랫폼 | PhysX 5(강체/관절), RTX 렌더러, USD 장면 |
| <b>Isaac Lab</b> | 로봇 학습 프레임워크 | 사전 정의 환경, 보상 함수, 커리큘럼 학습 |
| <b>RSL-RL</b> | RL 알고리즘 라이브러리 | PPO 구현, Actor-Critic 신경망 |
| <b>PyTorch</b> | 딥러닝 엔진 | GPU 텐서 연산, 자동 미분 |

## 왜 클라우드(AWS)인가?

| 항목 | 게이밍 PC (RTX 4090) | AWS g6e.4xlarge (L40S) |
|------|---------------------|----------------------|
| VRAM | 24 GB | <b>48 GB</b> |
| 동시 환경 수 | ~2,048 | <b>~4,096+</b> |
| 학습 시간 | ~2.5시간 | <b>~75분</b> |
| Docker 이미지 부담 | 로컬 디스크 22.6 GB | EBS로 자유 확장 |
| 비용 | GPU 구매 ~$2,000+ | 시간당 ~$3 (쓴 만큼만) |
| 스케일 | 고정 | 1 GPU ↔ 8 GPU 자유 전환 |
| 재현성 | 수동 설정 | <b>Terraform 코드로 동일 환경 재현</b> |
